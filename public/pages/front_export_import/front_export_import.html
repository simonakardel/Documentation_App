Export and Import in the front-end

In frontend JavaScript, the type="module" attribute is used to specify that a script should be treated as an ES module. With ES modules, you can use the import and export statements to modularize your code, allowing you to separate your code into smaller, reusable pieces.

1. Exporting

To export a variable, function, or class, use the export keyword. You can export multiple items from a single module. There are two types of exports: named exports and default exports.

Named exports

You can export multiple named items from a module. To create a named export, simply use the export keyword before the declaration of the variable, function, or class.

Default exports

Each module can have only one default export. A default export is usually used when a module has a primary item to export, such as a class or a single function.

2. Importing

To use the exported items from another module, use the import statement.

Importing named exports

When importing named exports, you need to use the same names as the ones used during exporting, and you must wrap them in curly braces {}.

Importing default exports

When importing a default export, you can choose any name you like, without the need for curly braces.
