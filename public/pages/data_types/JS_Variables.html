Data Types and Variables

1. Primitive Data Types

  • Number: Numeric values, including integers and floating-point numbers.
  • String: A sequence of characters enclosed in single or double quotes.
  • Boolean: Represents true or false values.
  • Null: Represents a non-existent or invalid object reference.
  • Undefined: Represents a variable that has not been assigned a value.

2. Non-Primitive Data Types

  • Object: A collection of properties, each with a key and value.
  • Array: A special kind of object used to store ordered collections of data.
  • Function: A reusable block of code that can be invoked with arguments.

3. Declaring Variables

In ES6 (ECMAScript 2015), there are three ways to declare variables: using var, let, and const. In modern JavaScript, it is recommended to use let and const over var due to their improved scoping behavior.

let

Variables declared with let are block-scoped, meaning their scope is limited to the block (enclosed by {}) in which they are declared. This prevents unintended access or modification of the variable outside its intended scope. let variables are not hoisted to the top of their scope, so they are only accessible after the line where they are declared. You can reassign values to let variables.


const

Variables declared with const are also block-scoped, like let. However, const variables must be assigned a value at the time of declaration, and their value cannot be changed afterward. This makes const suitable for declaring constants or values that should not be modified during the program's execution.

In summary, in modern JavaScript (ES6 and later), use let for variables that will change their values over time, and use const for variables that should remain constant. Both let and const provide better scoping behavior than var, which helps prevent bugs and makes your code easier to understand and maintain.

4. Scope

In JavaScript, scope refers to the accessibility and visibility of variables and functions within different parts of the code. It determines where a variable or function can be used and modified.

Global Scope

When a variable or function is declared outside of any function or block, it is said to be in the global scope. Global variables and functions can be accessed and modified from any part of the code, including within functions and blocks. However, using too many global variables can lead to conflicts and make your code harder to maintain.

Block Scope

When a variable is declared using the let or const keyword inside a block (enclosed by {}), it is said to be in the block scope. Block-scoped variables can only be accessed and modified within the block in which they are declared. They are not visible outside the block.
